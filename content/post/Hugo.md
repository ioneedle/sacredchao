### Проверка версии:

```
hugo version
```

### Обновление:

```
choco upgrade hugo
```

### Удаление:

```
choco uninstall hugo
```

## Инициализация простейшего шаблона статического сайта:

```
cd path_to_drectory
hugo new site site_name
```

Эта команда создаст директорию `site_name` в папке `path_to_drectory` со следующим содержимым:

```
.
├── archetypes
│   └── default.md
├── config.toml
├── content
├── data
├── layouts
├── static
└── themes

6 directories, 2 files
```

### Добавление темы:

```
cd themes
git clone https://github.com/carsonip/hugo-theme-minos.git themes/minos rm -rf themes/minos/.git
```

Такой способ установки (vendoring) в отличии от создания модуля (submodule) удобен тем, что тему в дальнейшем очень легко будет изменить под свои нужды и трансформировать в нечто совершенно иное.

Теперь необходимо изменить `config.toml` и указать какую тему мы хотим использовать. Добавьте в конец файла:

```
theme = "minos"
```

### Запуск сервера:

```
hugo server 
hugo server -D
```

Первая команда для сайта без черновиков публикаций, вторая \- с черновиками. Если сейчас зайти на `http://localhost:1313`, увидим сайт.

## Добавление контента

Находясь в корневой директории нашего проекта (`cd site_name` ), выполнить команду:

```
hugo new post/hello-world.md
```

Эта команда создаёт в директории `content` директорию `post`, а внутри неё файл `hello-world.md` со следующим содержимым:

```
--- 
title: "Hello World" 
date: 2020-03-07T17:25:04+03:00 
draft: true 
---
```

Часть, заключенная между `---`, называется front matter и содержит основную мета-информацию о статье. `title` будет использован при формировании списка статей и в качестве заголовка на странице статьи. `date` будет использована везде, где указывается дата написания статьи. `draft: true` означает, что статья не попадёт на сайт при публикации, но будет видна локально при построении сайта командой `hugo server -D` (`-D` означает построить черновики).

Изменим содержимое файла например так:

```
--- 
title: "Привет, мир!" 
date: 2020-03-07T17:25:04+03:00 
draft: true 
---

Пивет мир! Это первый пост на блоге, написанном на [Hugo](https://gohugo.io).
```

Заголовок в начале документа не дублируем. Зайдя теперь на локальную копию нашего сайта, увидим этот пост.

Hugo автоматически следит за изменениями в проекте и моментально перестраивает отображаемый сайт без необходимости даже обновлять страницу.

Давайте теперь добавим страницу `About`. Для этого выполним:

```
hugo new about.md
```

И изменим `content/about.md`:

```
--- 
title: "About" 
date: 2020-03-07T17:43:44+03:00 
draft: true 
hidden: true 
---

Это страница обо мне.
```

Свойство `hidden: true` позволит скрыть страницу из списка всех постов. Теперь необходимо добавить ссылку на неё в шапку сайта. Для этого допишите в конец `config.toml`:

```
[menu]

  [[menu.main]]
    identifier = "about"
    name = "About"
    url = "/about/"
    weight = -110
```

**Важно:** расположение контента (постов, страниц) и способ достичь одного и того же поведения (создание меню, сокрытие постов) зависят исключительно от используемой темы и могут кардинально отличаться от темы к теме. Кто-то может не поддерживать меню, заданное в конфиге, кто-то не позволит вам скрыть посты, кто-то потребует создавать страницы в чётких заранее определённых директориях.

## После публикации сайта

Исправим `config.toml`, указав в качестве `baseURL` полный адрес нашего сайта:

```
baseURL = "https://sample.com"
```

Единственная статья не отображается, так как она всё-ещё находится в состоянии черновика. Изменим `draft: true` на `draft: false`.

## Публикация на IPFS

В `config.toml` добавляем `relativeURLS` и устанавливаем  `true`.

```
relativeURLS=true
```

Формируем статичные страницы:

```
hugo -D
```

Страницы появятся в папке `./public/` , ее и надо залить на IPFS.
